IT-system testbed evaluation: CCD recovery sequence D_1 -> D_2 -> D_3
=====================================================================

Setup. Dockerized IT system with m = 10 web-service replicas n_1..n_10 behind a
load-balancing gateway, backed by a PostgreSQL database n_11; n_1 doubles as a
management host and is compromised (code execution). Detected attacker state
P~ = {P_0, P_1}. Functionality Phi = E[T] (total end-to-end throughput); critical
level alpha = 0.5 * Phi_nominal = 35.50 req/s (Phi_nominal = 71.01 req/s, the mean
over the 600-window nominal-operation dataset). CCD selects each mode from that
nominal dataset alone (causal inference via do-calculus; the degraded
configurations are confounded with low load in the data, so naive conditioning is
biased). Measurements: 100 windows of 6 s per mode on the live testbed; modes are
enacted with per-container iptables rules (REJECT with tcp-reset).

Mode D_1 -- containment (attack detected, nothing patched yet)
--------------------------------------------------------------
Intervention: do(N_1=0, M_1=0, A_2=0, ..., A_10=0)
  - N_1 = 0: close gateway -> n_1 (removes the compromised server's carried load
    T~_1 from T; the attacker controls T~_1 via P_1).
  - A_i = 0 (i = 2..10): close the management links n_1 -> n_i, blocking the
    lateral-movement exploits E_2..E_10.
  - M_1 = 0: close n_1 -> database, blocking the credential exploit E_11.
Together the closures block all 10 exploits reachable from P~, so the attacker
cannot gain any new privilege (containment criterion holds); N_1=0 disconnects
every attacker-controlled variable from the functionality variable T
(functionality criterion holds).
Inferred:  Phi-hat = 65.50 req/s = 92.2% of nominal (feasible: Phi-hat >= alpha).
Measured:  60.53 +/- 6.55 req/s = 93.1 +/- 10.1% of measured nominal
           (relative error of the inference: 7.6%).

Recovery action 1: patch the lateral-movement vulnerabilities on n_2..n_10 and
the database-credential vulnerability (removes exploits E_2..E_11 from the attack
graph Gamma). The attacker still holds P_1 (code execution on n_1).

Mode D_2 -- less restrictive (exploits patched, attacker still present)
-----------------------------------------------------------------------
Intervention: do(N_1=0)
With E_2..E_11 patched out of Gamma, containment is free: no exploit is feasible
from P~, so the management links A_2..A_10 and the database link M_1 are restored.
Only N_1 stays closed, because the attacker still controls n_1's carried load;
keeping n_1 out of the load-balancing pool keeps the attacker disconnected from T.
D_2 is strictly less restrictive than D_1 (D_1 superset D_2).
Inferred:  Phi-hat = 63.39 req/s = 89.3% of nominal (feasible).
Measured:  64.09 +/- 5.86 req/s = 98.6 +/- 9.0% of measured nominal
           (relative error 1.1%).

Recovery action 2: evict the attacker -- re-image n_1, which removes the foothold
vulnerability (patches E_1) and shrinks the detected state to P~ = {P_0}.

Mode D_3 -- full restoration (attacker evicted)
-----------------------------------------------
Intervention: do() (empty)
With the attacker evicted, no exploit is feasible and the derived set of
attacker-controlled variables is empty; both criteria hold with no closures, so
CCD returns the empty intervention and all links are restored.
Inferred:  Phi-hat = 68.21 req/s = 96.1% of nominal (feasible).
Measured:  70.56 +/- 6.30 req/s = 108.5 +/- 9.7% of measured nominal
           (relative error 3.4%).

Baselines (model-derived; shown in the inferred bar group only, since the attacker
software is not implemented on the testbed and neither baseline can be measured)
---------------------------------------------------------------------------------
- No degradation ("Attack"): the attack propagates fully -- the attacker gains
  code execution on every server and the database and zeroes every carried load
  T~_i, so T = sum_i N_i * T~_i = 0 exactly by the known functions F~.
  Phi_attack = 0.00 req/s = 0% of nominal (total loss of functionality).
- Naive containment ("Contain"): apply every blocking-edge closure regardless of
  functionality: do(M_1=0, A_2..A_10=0). Only M_1 has causal edges (the A_i are
  management links), so the impact is the loss of server 1's database access.
  Phi-hat = 64.19 req/s = 90.4% of nominal (estimated from the nominal dataset;
  cached in data/baseline_containment.json). Naive containment preserves less
  functionality than CCD's D_1 achieves after recovery relaxations (D_2/D_3) and,
  unlike CCD, never restores: it stays at 90.4% for the whole incident, and it
  leaves the attacker in control of T~_1 without the N_1 closure that CCD adds
  to guarantee the functionality criterion.

Notes
-----
- The mode sequence is monotone (D_1 superset D_2 superset D_3 = empty): each
  recovery action only removes attack-graph edges / shrinks P~, and CCD re-solves
  the same problem; the model, not the algorithm, encodes the recovery.
- % of nominal: inferred bars are normalized by Phi_nominal = 71.01 req/s (the
  nominal dataset mean); measured bars by the measured nominal run
  (65.01 +/- 7.14 req/s, 100 windows). Workload is random (W ~ U[50, 150] req/s
  per window), so per-run means fluctuate: D_2 and D_3 measuring slightly above
  their inferred values (and D_3 above 100%) is within the 95% confidence
  intervals, which overlap nominal for D_2/D_3.
- All three modes satisfy Phi >= alpha (= 50% of nominal) in both the inferred
  and the measured columns; the loss of functionality under containment is
  approximately 1/m, matching the analytic (m-1)/m = 90% at m = 10.
